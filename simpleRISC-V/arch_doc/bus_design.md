# SimpleRISC-V Bus 设计说明

## 1. 概述

本文档说明 `simpleRISC-V` 项目中共享总线模块的接口定义、内部逻辑和工作流程。

当前总线设计具有以下特征：

- 地址宽度为 32 位，采用字节地址。
- 数据宽度为 32 位。
- 支持 `4` 个主设备（Master）。
- 支持 `8` 个从设备（Slave）。
- 仲裁方式为固定优先级仲裁，编号越小优先级越高。
- 当前总线拥有者在请求保持有效时不会被中途抢占。
- 共享地址/控制/写数据总线，以及返回读数据/就绪信号，采用三态风格建模。
- 控制信号大量采用低有效约定。

总线相关源码位于：

- `src/bus/bus_top.v`
- `src/bus/bus_arbiter.v`
- `src/bus/bus_master_mux.v`
- `src/bus/bus_addr_dec.v`
- `src/bus/bus_slave_mux.v`

---

## 2. 顶层模块说明

顶层模块为 `bus_top`，负责把仲裁器、主设备侧共享总线、地址译码器和从设备返回通路连接起来。

模块功能划分如下：

- `bus_arbiter`：对多个 Master 的请求进行仲裁，输出授权。
- `bus_master_mux`：将获得授权的 Master 驱动到共享地址/控制/写数据总线上。
- `bus_addr_dec`：根据共享地址和地址选通信号生成从设备片选。
- `bus_slave_mux`：把被选中的 Slave 读数据和 ready 信号返回给 Master 侧。

---

## 3. 总线参数

默认参数如下：

| 参数名 | 默认值 | 说明 |
| --- | ---: | --- |
| `NUM_MASTERS` | `4` | Master 数量 |
| `NUM_SLAVES` | `8` | Slave 数量 |
| `ADDR_W` | `32` | 地址宽度，字节地址 |
| `DATA_W` | `32` | 数据宽度 |
| `IDX_MSB` | `31` | Slave 编号译码高位 |
| `IDX_LSB` | `29` | Slave 编号译码低位 |

在默认配置下，地址 `addr[31:29]` 用于选择从设备编号，共可映射 8 个 Slave。

---

## 4. 顶层 IO 说明

### 4.1 时钟与复位

| 信号名 | 方向 | 位宽 | 说明 |
| --- | --- | ---: | --- |
| `clk` | input | 1 | 系统时钟 |
| `rst_n` | input | 1 | 异步低有效复位 |

### 4.2 Master 侧接口

| 信号名 | 方向 | 位宽 | 说明 |
| --- | --- | ---: | --- |
| `m_req_n` | input | `NUM_MASTERS` | Master 请求总线，低有效 |
| `m_grnt_n` | output | `NUM_MASTERS` | 仲裁授权，低有效 |
| `m_addr` | input | `NUM_MASTERS * ADDR_W` | 各 Master 打包后的地址输入 |
| `m_as_n` | input | `NUM_MASTERS` | 地址选通，低有效 |
| `m_rw` | input | `NUM_MASTERS` | 读写控制，`1` 为读，`0` 为写 |
| `m_wr_data` | input | `NUM_MASTERS * DATA_W` | 各 Master 打包后的写数据 |
| `m_rd_data` | output | `DATA_W` | 共享返回读数据 |
| `m_rdy_n` | output | 1 | 共享 ready，低有效 |

说明：

- `m_addr` 与 `m_wr_data` 使用打包总线方式组织，每个 Master 占一个连续切片。
- 只有获得授权的 Master 会真正驱动共享总线。

### 4.3 Slave 侧接口

| 信号名 | 方向 | 位宽 | 说明 |
| --- | --- | ---: | --- |
| `s_addr` | output | `ADDR_W` | 共享地址总线 |
| `s_as_n` | output | 1 | 共享地址选通，低有效 |
| `s_rw` | output | 1 | 共享读写控制 |
| `s_wr_data` | output | `DATA_W` | 共享写数据总线 |
| `s_cs_n` | output | `NUM_SLAVES` | 各 Slave 片选，低有效 |
| `s_rd_data` | input | `NUM_SLAVES * DATA_W` | 各 Slave 打包后的读数据输入 |
| `s_rdy_n` | input | `NUM_SLAVES` | 各 Slave 的 ready 输入，低有效 |

说明：

- `s_cs_n` 仅在 `s_as_n` 有效时进行地址译码。
- 从设备返回通路也是共享的，只有被片选的 Slave 才能驱动返回总线。

## 5. 子模块逻辑说明

### 5.1 `bus_arbiter`

`bus_arbiter` 实现固定优先级仲裁。

仲裁规则如下：

1. `m0` 优先级最高，`m1` 次之，依次类推。
2. 当总线空闲时，仲裁器从低编号到高编号扫描请求信号。
3. 找到第一个有效请求后，立即输出授权。
4. 一旦某个 Master 成为当前拥有者，只要该 Master 的请求仍保持有效，就持续持有总线。
5. 当前拥有者释放请求后，仲裁器才重新扫描并选择新的拥有者。

这种策略的特点：

- 逻辑简单，延迟稳定。
- 有利于保证关键主设备的访问优先级。
- 低优先级设备在高负载下可能等待较久。

### 5.2 `bus_master_mux`

`bus_master_mux` 负责把获授权 Master 的输出连接到共享总线。

当前实现采用三态风格建模：

- 获授权 Master 驱动 `s_addr`、`s_as_n`、`s_rw`、`s_wr_data`。
- 未获授权 Master 对共享总线输出高阻态 `Z`。
- `s_as_n` 和 `s_rw` 使用 `tri1` 建模，因此在空闲时默认回到高电平。
- `s_addr` 和 `s_wr_data` 在空闲时为高阻态。

这样做的意义：

- 在仿真中更直观地表达“只有一个 Master 在驱动共享总线”。
- 当出现非法多驱动时，仿真更容易暴露冲突。

### 5.3 `bus_addr_dec`

`bus_addr_dec` 根据共享地址高位进行从设备选择。

默认规则如下：

- 使用 `addr[31:29]` 作为 Slave 编号。
- `000` 对应 `slave0`
- `001` 对应 `slave1`
- ...
- `111` 对应 `slave7`

译码同时受 `s_as_n` 控制：

- 当 `s_as_n = 0` 时，译码器根据地址生成唯一片选。
- 当 `s_as_n = 1` 时，所有 `s_cs_n` 保持无效。

这样可以避免总线空闲时因为地址线残留值或高阻态导致误片选。

### 5.4 `bus_slave_mux`

`bus_slave_mux` 负责将被片选 Slave 的响应返回给 Master 侧。

当前实现同样采用三态风格建模：

- 被片选 Slave 驱动 `m_rd_data` 与 `m_rdy_n`。
- 未片选 Slave 输出高阻态。
- `m_rdy_n` 使用 `tri1` 建模，因此空闲时默认回到高电平。
- `m_rd_data` 在空闲时为高阻态。

## 6. 读写事务流程

### 6.1 读事务

读事务的基本流程如下：

1. 某个 Master 拉低自己的 `m_req_n` 发起总线请求。
2. 仲裁器输出对应的 `m_grnt_n`。
3. 获授权 Master 通过共享总线驱动：
   - `s_addr`
   - `s_as_n`
   - `s_rw = 1`
4. 地址译码器根据 `s_addr[31:29]` 产生对应 `s_cs_n`。
5. 被选中的 Slave 返回：
   - `s_rd_data`
   - `s_rdy_n`
6. `bus_slave_mux` 将选中 Slave 的响应送回 `m_rd_data` 与 `m_rdy_n`。
7. Master 在 ready 到达后完成本次读访问。

### 6.2 写事务

写事务流程与读事务类似，区别在于：

- `s_rw = 0`
- `s_wr_data` 由获授权 Master 驱动到共享总线
- Slave 根据地址、片选和写数据完成写入

---

## 7. 三态建模说明

本设计在共享总线位置使用三态风格建模，主要用于表达总线所有权和总线空闲状态。

需要注意：

- 在大多数 FPGA 综合流程中，内部三态总线通常不会被综合成真正的内部三态结构。
- 综合工具一般会把内部三态网络折叠为多路选择器逻辑。
- 因此，这里的三态更适合作为总线行为建模和仿真表达。
- 如果未来需要真实的外部三态总线，建议把真正的三态控制放在芯片顶层或 IO Buffer 附近。

---

## 8. 有效电平约定

本总线大量采用低有效控制信号，使用时需要特别注意：

| 信号 | 有效电平 | 说明 |
| --- | --- | --- |
| `rst_n` | 低有效 | 复位 |
| `m_req_n` | 低有效 | Master 请求 |
| `m_grnt_n` | 低有效 | 仲裁授权 |
| `m_as_n` | 低有效 | 地址选通 |
| `s_as_n` | 低有效 | 共享地址选通 |
| `s_cs_n` | 低有效 | Slave 片选 |
| `s_rdy_n` | 低有效 | Slave ready |
| `m_rdy_n` | 低有效 | Master 侧 ready |

## 9. 当前验证覆盖

当前已编写简单定向测试 `dv/bus/bus_top_tb.v`，覆盖以下场景：

- 单个 Master 连续访问多个 Slave
- 写数据路径检查
- 多个 Master 同时请求时的固定优先级仲裁
- 当前总线拥有者在请求未释放时不被抢占
- 当前拥有者释放后重新按照优先级仲裁
- 总线空闲时三态与默认返回行为检查

---

## 10. 设计总结

该总线设计适合当前 simpleRISC-V 项目的早期原型阶段，优点是：

- 结构清晰
- 容易扩展参数
- 仲裁逻辑直接
- 仿真行为明确

后续如果项目继续演进，可以考虑增加以下能力：

- 超时与错误响应
- 突发传输
- 可配置优先级或轮询仲裁
- 总线锁定
- 地址对齐与访问保护检查
- 更完整的验证环境与随机测试
